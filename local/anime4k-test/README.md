# Локальный стенд Anime4K

Тест **апскейла 720p → ~2×** в браузере. Не для продакшена — только на вашем ПК.

## Быстрый старт

1. Положите видео или используйте готовые файлы:

   ```
   media/sample-1080.mp4   ← 1080p для браузера (лучший вариант)
   media/sample.mkv        ← только исходник; в Chrome не играет (EAC3)
   ```

   **MKV не работает в браузере** (звук EAC3 5.1). Конвертация:

   ```bash
   npm run remux        # первые 5 мин → sample-1080.mp4
   npm run remux:full   # весь фильм (~1.5 ч, большой файл)
   ```

2. В папке `local/anime4k-test`:

   ```bash
   npm install
   npm start
   ```

3. Откройте в браузере:

   ```
   http://localhost:8765
   ```

## Управление

| Действие | Клавиша |
|----------|---------|
| Play / Pause | `Space` |
| Anime4K вкл/выкл | `U` |
| Сравнение исходник / Ultra | `C` |

- **Anime4K** — WebGL-апскейл (профили Simple S/M/L и др.)
- **Сравнение A/B** — слева исходник, справа обработка
- В статистике: разрешение источника, canvas, FPS

## Требования

- Chrome / Edge (рекомендуется)
- Видеокарта с WebGL
- MP4 (H.264) — самый надёжный формат для `<video>`

## Заметки

- `vendor/anime4k.js` создаётся после `npm install` (~9 MB)
- Видеофайлы в `media/` не коммитятся в git
- Это **не настоящий 4K-файл**, а улучшение картинки на лету

## Максимальное качество (чеклист)

1. **Источник 1080p** — не 720p; по возможности файл с **высоким битрейтом** (не сильно сжатый).
2. **Профиль в плеере:** `Higher-end C ★ MAX` (по умолчанию).
3. **Полноэкранный режим** на мониторе 2K/4K — там разница видна сильнее всего.
4. **Chrome / Edge** на ПК с дискретной видеокартой.
5. Если FPS проседает (<30) — попробуй `Higher-end A` или `Simple UL`.

Формула: **лучший исходник + тяжёлый профиль Anime4K + большой экран**.
